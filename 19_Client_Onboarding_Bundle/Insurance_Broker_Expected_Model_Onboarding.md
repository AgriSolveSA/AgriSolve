# Onboarding Addendum — Expected Commission Model

To calculate leakage (Expected vs Actual), we need one of:

## Option A (Best): Policy register includes Premium
Provide:
- PolicyNumber, Provider, ProductType, Adviser
- Monthly premium (or annual premium + tell us which)
- Status and start date

Then we apply the rules-based rate table.

## Option B: You already have an expected commission export
Provide:
- ExpectedCommission_YYYYMM.csv with policy-level expected amounts
- We still validate with coverage and reconciliation.

## What happens first
Week 1: Ongoing expected model only (simple and reliable)
Week 2+: Add initial expected if data supports it.
